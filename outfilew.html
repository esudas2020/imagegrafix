‚öôÔ∏è Phase 3: Backend Development

1. Create the Application Logic (app.py)

Create app.py. This is the brain of your application. - Step 1: Imports & Config - Import Flask, sqlite3, render_template, etc. - Set a secret_key for session management. - Step 2: Database Setup - Write a function init_db() to create a users table with id, username, email, and password. - Step 3: Routes - /: Render index.html. - /register (POST): Insert new user into the database. Handle potential errors (e.g., duplicate email). - /login (POST): Check if email and password match a database record. - /admin: Fetch all users and render admin.html. - /delete/<id> (POST): Delete a user by ID.

üöÄ Phase 4: Run & Test

  1. Initialize the Database: Run the app once to create the database:

    python app.py

    (Ensure init_db() is called in if __name__ == "__main__":)

  2. Access the App: Open your browser and go to http://127.0.0.1:5000.

  3. Test Scenarios:

    • Register a new user.
    • Try logging in with incorrect credentials (should verify error handling).
    • Log in with correct credentials.
    • Go to /admin and try managing users.

üõ°Ô∏è Phase 5: Security Challenge (Advanced)

The base implementation has critical security flaws. Your next task is to fix them:

  1. Stop Storing Plain Text Passwords:
    • Problem: Passwords are saved as normal text in users.db.
    • Solution: Use werkzeug.security to hash passwords.
      • generate_password_hash(password) when registering.
      • check_password_hash(hash, password) when logging in.
  2. Secure the Admin Page:
    • Problem: Anyone can go to /admin.
    • Solution: Add a check. Only allow access if a user is logged in (use session).
  3. Protect Against CSRF:
    • Problem: Forms can be submitted by malicious sites.
    • Solution: Use Flask-WTF forms or add CSRF tokens manually.
